slamlab_offline

The slamlab_offline section configures the slamlab-offline executable: input point cloud directory, point cloud range, and optional sensor data files.

All paths support the $CONFIG_DIRPATH$ token, which is replaced with the parent directory of the config file. The $STAMP$ token is replaced with a YYYY_MM_DD-HH_MM_SS timestamp.

ParameterTypeDefaultDescription
clouds_dirpathpath""Directory containing .ply point cloud files. Point clouds are sorted alphabetically for processing order. Required.
first_cloud_idxint0Index of first point cloud to process (0-based into sorted list). Must be ≥ 0.
last_cloud_idxint-1Index of last point cloud to process. -1 means process all remaining point clouds.
imu_measurements_filepathpath""Path to IMU measurements file. If non-empty, IMU data is loaded and added to the pipeline.
absolute_position_measurements_filepathpath""Path to absolute position measurements file (e.g. GNSS). If non-empty, measurements are loaded and added to the pipeline.
