slamlab_core

The slamlab_core section configures the SLAM pipeline: point cloud preprocessing, matching, local map, ICP registration, trajectory estimation, and output artifacts.

clouds

Section titled “clouds”

The following parameters apply to the original incoming point clouds.

distance_filter

Section titled “distance_filter”
ParameterTypeDefaultDescription
enabledboolfalseFilter points by distance from sensor origin
min_distancefloat0.0Minimum distance in meters
max_distancefloat.infMaximum distance in meters

intensity_filter

Section titled “intensity_filter”
ParameterTypeDefaultDescription
enabledboolfalseFilter points by intensity
min_intensityfloat0.0Minimum intensity
max_intensityfloat.infMaximum intensity

profile_filter

Section titled “profile_filter”
ParameterTypeDefaultDescription
enabledboolfalseFiltering of rows and columns of an organized point cloud
row_intervaluint321Sample every Nth row
col_intervaluint321Sample every Nth column

For unorganized point clouds, the cloud is interpreted as N x 1 (N rows, 1 column). In that case, row_interval performs index-based subsampling (for example, row_interval=2 keeps every second point), while col_interval is ignored.

voxel_filter

Section titled “voxel_filter”
ParameterTypeDefaultDescription
enabledboolfalseVoxel-based downsampling - keeps first point in each voxel
voxel_sizefloat0.1Voxel grid size in meters

normal_estimation

Section titled “normal_estimation”
ParameterTypeDefaultDescription
enabledboolfalseEstimate surface normals for organized point clouds using a local neighborhood
kernel_widthuint323Kernel width (must be odd, ≥ 3)
kernel_heightuint323Kernel height (must be odd, ≥ 3)
max_roughnessfloat0.2Max roughness to accept a normal

voxel_planes

Section titled “voxel_planes”
ParameterTypeDefaultDescription
voxel_sizefloat1.0Voxel size for plane extraction in meters
min_planarityfloat0.0Minimum planarity (0 = line-like, 1 = plane)
max_roughnessfloat0.1Maximum roughness to accept as plane
max_num_points_per_planeuint321000Max points per plane (≥ 100)

matching

Section titled “matching”
ParameterTypeDefaultDescription
cloud_to_local_mapbooltrueMatch incoming point clouds against local map
cloud_to_prior_mapboolfalseAlso match against a given prior map

At least one must be true (unless IMU measurements are provided).

local_map

Section titled “local_map”
ParameterTypeDefaultDescription
map_growsbooltrueMap grows over time; if false, map resets each iteration
map_radiusfloat20.0Radius around current pose to keep point clouds (meters, 0 = no limit)
min_distance_between_cloud_originsfloat1.0Min distance before adding a new point cloud to the map (meters)

voxel_planes

Section titled “voxel_planes”
ParameterTypeDefaultDescription
min_planarityfloat0.25Minimum planarity for plane detection
max_roughnessfloat0.15Maximum roughness
min_num_points_per_planeuint321Minimum points per plane
max_num_points_per_planeuint32100Maximum points per plane

prior_map

Section titled “prior_map”
ParameterTypeDefaultDescription
filepathstring""Path to external voxel planes map file (.vpl)
min_distance_between_cloud_originsfloat5.0Only re-match after moving this distance (meters)

Only used when matching.cloud_to_prior_map is true. If enabled, filepath must not be empty.

icp

Section titled “icp”
ParameterTypeDefaultDescription
max_iterationsuint325Max ICP iterations per point cloud
matching_methodstring"same_voxel"same_voxel or nearest_neighbor
loss_functionstring"trivial_loss"Robust loss function (see below)
max_point_to_point_distancefloat1.0Reject point pairs exceeding this distance (meters)
max_point_to_plane_distancefloat0.4Reject plane correspondences exceeding this distance (meters)
max_angle_between_normalsfloat45.0Reject pairs with normals differing more than this (degrees)
sigma_mad_limitfloat4.0Outlier threshold (median absolute deviation)
correspondences_max_weightfloat1000.0Maximum weight per correspondence

matching_method options:

  • same_voxel — fast, uses voxel grid structure for correspondences
  • nearest_neighbor — slower KD-tree based, more flexible

loss_function options: trivial_loss, huber_loss, soft_l_one_loss, cauchy_loss, arctan_loss, tukey_loss

loop_closure_detection

Section titled “loop_closure_detection”
ParameterTypeDefaultDescription
enabledboolfalseEnable loop closure detection
filter_key_cloudsbooltrueApply preprocessing filters to key point clouds
min_distance_between_key_cloud_originsfloat1.0Min distance between key point cloud origins (meters)
dirpathstring""Directory for key point clouds and results (required if enabled)

matching_of_key_clouds

Section titled “matching_of_key_clouds”
ParameterTypeDefaultDescription
max_distancefloat15.0Max distance between origins to consider as loop (meters)
min_time_differencefloat20.0Min time between key point clouds for loop closure (seconds)

Caution

Cannot be enabled simultaneously with trajectory.fixed_lag_smoother.

trajectory

Section titled “trajectory”

Section titled “fixed_lag_smoother”
ParameterTypeDefaultDescription
enabledboolfalseUse fixed-lag smoother instead of full batch optimization

Cannot be enabled simultaneously with loop_closure_detection.

Section titled “imu (optional)”
ParameterTypeDefaultDescription
accelerometer_sigmafloat1.0Accelerometer noise std dev (m/s²)
gyroscope_sigmafloat1.0Gyroscope noise std dev (rad/s)
integration_sigmafloat0.0Integration error std dev
accelerometer_bias_sigmafloat1.0Accelerometer bias random walk (m/s³)
gyroscope_bias_sigmafloat1.0Gyroscope bias random walk (rad/s²)

acceleration_unit_is_gboolfalseIf true, accelerometer data is in g units

Section titled “absolute_position_measurements (optional)”

min_elapsed_time_since_last_added_measurement_secfloat0.5Time gate between measurements (seconds)
min_distance_to_last_added_measurementfloat5.0Distance gate between measurements (meters)

static_transforms

Section titled “static_transforms”

Array of sensor-to-base transforms:

ParameterTypeDefaultDescription
framestring"lidar"Sensor frame name
base_T_frame.translationfloat[3][0, 0, 0]Translation x, y, z (meters)
base_T_frame.rotationfloat[4][0, 0, 0, 1]Quaternion qx, qy, qz, qw

initial_base_pose

Section titled “initial_base_pose”
ParameterTypeDefaultDescription
translationfloat[3][0, 0, 0]Starting position x, y, z (meters)
rotationfloat[4][0, 0, 0, 1]Starting orientation qx, qy, qz, qw

artifacts

Section titled “artifacts”
ParameterTypeDefaultDescription
enabledbooltrueSave trajectory and optional point cloud artifacts
dirpathstring""Output directory (required if enabled)

save_clouds

Section titled “save_clouds”
ParameterTypeDefaultDescription
enabledboolfalseSave individual registered point clouds to disk
generate_prior_mapboolfalseGenerate a prior map from saved point clouds

When enabled, the following filters can be applied independently to saved point clouds (same structure as clouds filters above):

  • distance_filter — {enabled, min_distance, max_distance}
  • intensity_filter — {enabled, min_intensity}
  • profile_filter — {enabled, row_interval, col_interval}
  • normal_estimation — {enabled, kernel_width, kernel_height}

tiling

Section titled “tiling”
ParameterTypeDefaultDescription
enabledboolfalseSplit saved point clouds into spatial tiles
tile_sizeint[3][50, 50, 50]Tile size in x, y, z (meters)
voxel_filter.enabledboolfalseApply voxel filter to tiles
voxel_filter.voxel_sizefloat0.1Voxel size for tile downsampling (meters)

debug_mode

Section titled “debug_mode”
ParameterTypeDefaultDescription
enabledboolfalseSave intermediate processing stages
dirpathstring""Debug output directory (required if enabled)

send_visualization_data

Section titled “send_visualization_data”
ParameterTypeDefaultDescription
enabledbooltrueSend real-time data to slamlab-visualization
receiver_ip_addressstring"127.0.0.1"IP address of visualization client
